# 本草问方视觉设计与约束文档 V2

## 1. 文档目标
- 患者端为移动端风格
- 医生端为 Web 平台
- 本版仅保留纯视觉设计与界面约束内容
- 不展开流程、技术、前端架构与实现细节

本文档用于统一：

- 品牌视觉基调
- 配色、字体、间距、圆角、阴影
- 患者端与医生端的视觉差异
- 组件、图标、插画、图表的使用规则
- 样例图风格的复现要点

## 2. 已采用的默认设计决策

本文档默认采用以下决策：

1. 不做深色模式
2. 患者端与医生端共用一套品牌色体系
3. 医生端在同品牌体系下，整体饱和度更低、更克制
4. 患者端允许更多插画、情绪化表达和柔和装饰
5. 医生端减少装饰，强调结构化信息和专业感
6. 统一建设一套按钮、卡片、标签、输入框、状态条组件库

## 3. 品牌名称使用规则

### 3.1 当前默认产品名称

当前默认品牌名为：

`本草问方`

### 3.2 名称可切换约束

1. 品牌图标与品牌文字分离设计
2. 不把“本草问方”四个字直接做进不可拆分插画或背景图中
3. 标题区、欢迎页、登录页、导航栏品牌位必须采用可替换文本布局
4. 品牌名称长度变化后，布局不能崩坏
5. 按钮、头图、空状态图中不得出现写死的品牌文案图片

### 3.3 品牌位设计建议

- 用占位图

## 4. 品牌视觉定位
### 4.2 建议整体气质

本项目的视觉不应做成：

- 纯互联网后台
- 纯医院蓝医疗系统
- 重装饰国潮风
- 过度科技感 AI 产品

建议做成：

`现代草本医疗风`

即：

- 有中医药感，但不厚重
- 有智能感，但不科幻
- 有医疗可信度，但不压迫

## 5. 视觉风格总原则

### 5.1 核心原则

1. 整体色调以绿色体系为中心
2. 大圆角、大留白、大卡片
3. 用白色卡片承载内容
4. 用低饱和中性色保证阅读舒适
5. 装饰性只服务品牌氛围，不能干扰信息阅读

### 6.1 必须复现的核心视觉特征

1. 高识别度亮绿色品牌主色
2. 深绿色作为导航、图标底和稳定视觉锚点
3. 白色圆角卡片作为主要信息容器
4. 顶部大色块头图或大圆角头部结构
5. 大尺寸圆角输入框与按钮
6. 患者端底部深色胶囊导航
7. 大量留白和松弛的卡片间距
8. 少量但明确的健康数据图形表达

### 6.2 患者端样例图复现清单

- 首页要有明显的绿色品牌头部
- 搜索框、标签、内容卡片都要明显圆润
- 卡片信息密度不能过高
- 主按钮必须足够醒目、足够大
- 登录页和欢迎页要有较强品牌识别
- 个人页允许使用绿色植物感头图

### 6.3 医生端转译清单

医生端不直接照搬样例图，但必须继承其品牌基因：

- 继承绿色体系
- 继承白卡片承载信息
- 继承圆角和柔和阴影
- 降低亮绿色面积
- 增加结构化分区
- 保持更高密度但不失呼吸感

### 6.4 不能偏离的地方

- 不能改成科技蓝主色
- 不能改成传统后台灰蓝风
- 不能改成重中式国潮风
- 不能把圆角、留白、卡片感削弱掉

## 7. 配色系统

### 7.1 主品牌色

#### 品牌主绿

- `#84D60D`
- 用于：主按钮、品牌标题高亮、核心入口、重点提示

#### 品牌深绿

- `#2F4538`
- 用于：导航、标题、深色底栏、重点图标底色

#### 草本中绿

- `#6FBF2A`
- 用于：标签、次级按钮、浅强调卡片

### 7.2 背景色

| 用途 | 色值 |
| --- | --- |
| 主背景 | `#F7F5EF` |
| 页面浅背景 | `#F3F4EE` |
| 卡片背景 | `#FFFFFF` |
| 分区底色 | `#EEF5E8` |

### 7.3 中性色

| 语义 | 色值 |
| --- | --- |
| 标题文字 | `#24312C` |
| 正文文字 | `#3E4A45` |
| 次级文字 | `#6C756F` |
| 占位文字 | `#9CA39E` |
| 分割线 | `#E8ECE6` |

### 7.4 功能色

| 语义 | 色值 | 说明 |
| --- | --- | --- |
| 成功 | `#59A96A` | 成功、健康、完成状态 |
| 警告 | `#E5AE39` | 注意、提醒 |
| 错误 | `#E86C76` | 错误、风险、表单校验 |
| 信息 | `#6C92B5` | 普通提示、辅助状态 |

### 7.5 中医药题材点缀色

| 名称 | 色值 | 用途 |
| --- | --- | --- |
| 药黄 | `#F3D46B` | 小标签、身份标识 |
| 草木粉 | `#F7E7EA` | 柔和数据卡底 |
| 药蓝灰 | `#DDEAF3` | 图表与辅助数据区 |

### 7.6 配色约束

- 亮绿不能大面积铺满医生端主工作区
- 白色卡片必须作为主要信息承载面
- 红色只用于错误和风险
- 黄色只作为辅助高亮，不作为核心品牌色
- 患者端可以用更亮的品牌绿
- 医生端应优先使用深绿、灰白、浅绿

## 8. 字体与层级

### 8.1 字体建议

- 中文：`PingFang SC`, `HarmonyOS Sans SC`, `Noto Sans SC`
- 英文与数字：沿用同一无衬线体系

### 8.2 字体气质要求

- 简洁
- 圆润但不幼稚
- 数字清晰
- 标题稳重

### 8.3 字号建议

#### 移动端

- 一级标题：28-32
- 二级标题：22-24
- 模块标题：18-20
- 正文：14-16
- 辅助信息：12-13

#### 医生端 Web

- 页面标题：28-32
- 分区标题：20-24
- 卡片标题：16-18
- 表格正文：14-15
- 辅助信息：12-13

### 8.4 字体使用约束

- 不使用书法体作为界面主字体
- 不使用过细的浅灰小字
- 长文本必须保证足够行高
- 标题与按钮文案尽量简短有力

## 9. 圆角、间距、阴影

### 9.1 圆角

| 元素 | 圆角建议 |
| --- | --- |
| 页面大卡片 | 24-32px |
| 普通卡片 | 20-24px |
| 输入框 | 18-24px |
| 主按钮 | 18-24px |
| 小标签 | 胶囊形 |
| 弹窗 | 24-28px |

### 9.2 间距系统

建议使用 8px 基础栅格：

- 4
- 8
- 12
- 16
- 24
- 32
- 40

### 9.3 阴影建议

- `0 6 24 rgba(47, 69, 56, 0.08)`
- `0 2 10 rgba(47, 69, 56, 0.06)`

阴影要求：

- 要轻
- 要软
- 不要发光感

## 10. 患者端视觉规范

### 10.1 总体方向

患者端采用：

`草本友好型移动医疗界面`


### 10.3 患者端可使用的视觉手法

- 绿色头图区
- 插画式欢迎页
- 大圆角卡片
- 大按钮
- 轻数据卡片
- 情绪友好的提示语

### 10.4 患者端约束

- 页面简单

## 11. 医生端 Web 视觉规范

### 11.1 总体方向

医生端采用：

`低饱和草本专业 Web 工作台`

### 11.2 页面气质

- 更稳重
- 更克制
- 更理性
- 更专业

### 11.3 医生端布局建议

- 左侧导航 + 顶部操作栏
- 内容区以卡片与分区块为主
- 可采用双栏或三栏结构
- 主信息区宽度优先保证阅读与数据浏览效率

### 11.4 医生端视觉约束

- 不使用大面积亮绿背景
- 不使用过多插画装饰
- 不做患者端那种情绪化卡片
- 不用过大的圆角导致页面显得幼态
- 表格和卡片必须保持统一节奏


## 12. 组件视觉规范

### 12.1 按钮

- 主按钮：品牌绿底、白字
- 次按钮：白底绿边或浅绿底
- 危险按钮：白底红边或浅红底
- 按钮高度：
  - 移动端建议 44px 起
  - Web 建议 36px / 40px / 48px 分级

### 12.2 输入框

- 高圆角
- 白底
- 轻边框
- 聚焦态使用浅绿边框或柔和外发光
- 错误态使用浅红描边与错误文本

### 12.3 卡片

- 卡片是主视觉单元
- 一张卡片只承载一个主内容
- 卡片内必须有明确标题、内容、操作区层级
- 卡片内部留白不可过小

### 12.4 标签

- 胶囊标签优先
- 颜色要浅，不要太花
- 标签主要用于状态、分类、身份

### 12.5 导航

#### 患者端

- 底部胶囊导航可保留
- 不超过 5 个一级入口

#### 医生端

- 左侧导航更适合 Web
- 选中态用深绿背景或浅绿高亮条

## 13. 图标、插画与图表

### 13.1 图标

- 圆角线性图标优先
- 图标线条不要太尖锐
- 图标风格要统一，不混用多套

### 13.2 插画

插画只建议在以下区域出现：

- 欢迎页
- 登录页
- 空状态页
- 引导页

插画风格要求：

- 几何简化
- 扁平柔和
- 少细碎装饰
- 少厚重传统纹样

### 13.3 图表

- 图表应简洁
- 控制颜色数量
- 用品牌绿 + 药蓝灰 + 草木粉即可
- 不做炫技型动态图表

## 14. 页面装饰策略

### 14.1 可以使用的装饰元素

- 叶片纹理
- 柔和波形
- 草本抽象图形
- 极轻的浅色背景纹理

### 14.2 不建议使用的装饰元素

- 重国潮纹样
- 大面积祥云、卷轴、印章
- 黑金、红金古风搭配
- 高复杂度背景图

### 14.3 装饰使用原则

- 装饰只用于建立气质
- 装饰不能进入主内容阅读区中心
- 装饰不应干扰表单、列表和图表阅读

## 15. 图片与人物素材使用规则

### 15.1 患者端

- 可使用真实头像
- 可使用轻插画或品牌角色
- 可使用自然植物局部作为头图背景

### 15.2 医生端

- 医生头像建议真实、专业、清晰
- 不建议大量使用素材库“摆拍医生照”
- 图像更多用于身份识别，不用于装饰

## 16. 无障碍与可用性约束

### 16.1 颜色对比

- 浅绿底白字必须检查可读性
- 次文字不能浅到难读
- 状态不能只靠颜色区分

### 16.2 点击区域

- 移动端可点击区域至少 44px
- Web 端操作按钮不能过密

### 16.3 表单可用性

- 错误信息必须明确
- 占位符不能替代标签
- 必填项、禁忌项、风险项需要显著

## 17. 禁区项

本项目视觉上明确不建议做以下事情：

1. 用科技蓝替换整套绿色品牌体系
2. 大面积使用高饱和亮绿覆盖医生端工作台
3. 用古风大红金色做主界面
4. 大量书法字、篆刻字上界面
5. 将患者端和医生端做成完全同一种信息密度
6. 同一页面中同时出现太多色块、太多阴影、太多装饰
7. 过度卡通化

## 18. 最终设计结论

### 18.1 品牌结论

当前统一使用品牌名：

`本草问方`

但视觉系统必须保证后续在不改动核心样式语言的前提下，可以替换为新的品牌名称。

### 18.2 视觉推荐结论

本项目的最佳视觉方向是：

`绿色草本医疗风 + 柔和圆角卡片系统 + 患者端轻友好 / 医生端低饱和专业化`

### 18.3 一句话标准

如果最后做出来的页面看起来像：

- 干净
- 温和
- 有中医药气质
- 不古风
- 不科技炫技
- 患者端亲和
- 医生端专业

并且能够稳定复现样例图中的绿色头部、白卡片、大圆角、深色导航和轻健康感，那么这套视觉方向就是对的。

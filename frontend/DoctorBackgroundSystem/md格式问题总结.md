# Markdown 渲染问题处理简述

## 问题现象

智能助手流式返回 Markdown 内容时，页面出现以下问题：

- 标题未被识别，例如 `### 主治` 显示成普通文本
- 列表没有正确换行，例如 `- 大补` 和上一段内容黏在同一行
- 有些分段内容被拼成一整段，导致 Markdown 结构失效

## 排查结论

本次问题的核心不在 Vue 组件本身，而在流式分片处理过程中，Markdown 原始文本边界被破坏了。

主要原因有三点：

1. `response` 分片曾被执行过 `.trim()`，导致前后空格和换行丢失。
2. 后端返回中既有真实换行，也可能有字面量 `\\n`，前端如果不统一处理，就会出现段落粘连。
3. 部分标题和列表格式不标准，例如 `###主治`、`-大补`，会降低当前渲染器的识别率。

## 处理过程

1. 保留流式返回的 `response` 原文，不再对正文做 `.trim()`。
2. 在渲染前统一换行格式，将 `\r\n`、`\r` 和 `\\n` 归一化为 `\n`。
3. 调整消息展示逻辑，按累计后的完整文本重新渲染 Markdown，而不是把每个分片单独当作独立块处理。
4. 对标题格式做最小兼容，例如把 `###主治` 识别为 `### 主治`。

## 当前结论

这次问题本质上是“Markdown 文本在流式拼接阶段被破坏”，不是单纯的样式问题。

后续如果再出现类似情况，优先检查：

- 流式分片是否丢失换行
- 是否对 `response` 做了 `.trim()` 或其他清洗
- 渲染前是否把 `\\n` 正确还原成真实换行

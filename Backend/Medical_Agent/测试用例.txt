================================================================================
                      中医药两级智能诊疗Agent项目测试用例
================================================================================

测试环境：
- 前端：Uni-app微信小程序（患者端）+ Vue3 PC医生网页后台
- 后端：FastAPI + MySQL + Neo4j知识图谱
- AI引擎：LangGraph两级调度Agent + 阿里云DashScope API（qwen-max/qwen-turbo）
- 业务模式：线下面诊 + 线上挂号，患者端与医生端分别独立与Agent对话，不存在医患在线聊天界面

测试用例编号规则：
- P_XXX：患者端测试用例
- D_XXX：医生端测试用例
- A_XXX：AI Agent测试用例
- B_XXX：后端API测试用例
- KG_XXX：知识图谱测试用例
- I_XXX：集成测试用例

【P_001】患者注册建档 - 正常流程
模块：账号注册&标准化电子病历建档
优先级：高
前置条件：用户未注册，微信授权可用
测试步骤：
1. 打开小程序，点击"微信授权登录"
2. 系统强制跳转至建档页面
3. 填写完整信息：姓名、性别、年龄、手机号、身份证、过敏史、既往慢病、家族病史
4. 点击"提交建档"
预期结果：
1. 成功获取微信授权信息
2. 强制进入建档页面，不完成无法使用其他功能
3. 表单验证通过，提交成功
4. 返回提示"建档成功"，生成唯一patient_id
备注：检查patient_id是否全局唯一

3. 仅填写姓名，其他为空，点击"提交"
4. 身份证号填写非法格式（如12位），点击"提交"
5. 手机号填写非11位数字，点击"提交"
预期结果：
1. 提示"请填写姓名"
2. 提示"请填写性别"、"请填写年龄"等必填项
3. 提示"身份证号格式不正确"

【P_004】AI预问诊 - 正常多轮对话
模块：AI预问诊&导诊挂号模块
优先级：高
前置条件：患者已注册建档，无过敏史
测试步骤：
1. 进入AI预问诊页面
2. 用户输入："我最近总是头晕、腹痛"
3. Agent追问："请问您的舌象是怎样的？"
4. 用户输入："舌苔薄白"
5. Agent追问："请问您的脉象是怎样的？"
6. 用户输入："脉浮紧"
预期结果：
1. 进入预问诊对话页面，显示免责声明
2. Agent识别症状并追问缺失信息（舌象）
3. 用户回答后，Agent继续追问缺失信息（脉象）
4. 信息完整后，Agent输出辨证结论和推荐科室
5. 显示"仅供参考，不可替代面诊"免责声明

【P_005】AI预问诊 - 追加提问模式
模块：AI预问诊&导诊挂号模块
优先级：高

预期结果：
1. 进入追加提问模式，保留原有症状信息
2. Agent锁定已有信息，基于新增症状"流鼻涕"拓展咨询
3. 不重复追问已收集的舌象、脉象信息
4. 更新辨证结果，可能调整推荐科室

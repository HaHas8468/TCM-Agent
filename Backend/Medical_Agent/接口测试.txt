============================================================
中医药诊疗智能体 API 接口测试用例（Postman）
============================================================

测试环境：http://localhost:8000

说明：
- 所有需要登录的接口需携带 Authorization: Bearer <token>
- 响应格式统一：成功 { "code": 0, "data": {...} }，失败 { "code": 非0, "msg": "..." }
- 时间字段格式：ISO8601（YYYY-MM-DDTHH:mm:ss）

============================================================
一、鉴权相关（3个接口）
============================================================

【1】患者注册
POST /api/auth/register
Headers:
    Content-Type: application/json

Body:

    "password": "12345678",
    "confirm_password": "12345678"
}

============================================================
测试流程建议
============================================================

1. 先注册患者账号 → 登录获取患者token
2. 登录医生账号获取医生token
3. 查询科室列表 → 获取医生ID
4. 查询医生可预约时段
5. 创建挂号订单
6. 医生查看接诊队列
7. 医生开始接诊 → 提交四诊信息 → 完成接诊
8. 患者查看历史诊断

============================================================
Postman环境变量建议
============================================================

在Postman中创建环境变量：
- base_url: http://localhost:8000
- patient_token: {{患者登录后获取的token}}
- doctor_token: {{医生登录后获取的token}}

Authorization header使用：
Bearer {{patient_token}} 或 Bearer {{doctor_token}}
